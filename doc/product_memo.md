# Aligner – 内在对齐引擎（减脂版）

## 一、产品定位

Aligner 是一套基于教练协议的长期行为对齐系统。

减脂是第一落地场景，但产品核心并非"管理卡路里"，而是训练用户在**状态（State）—意图（Intention）—行为（Action）**之间建立一致性。

它填补效率工具与心理疗愈之间的真空：不追求外在生产力优化，也不进行病理式治疗，而是解决"内在动力阻滞"。

核心问题不是"吃多少"，而是：**为什么在明知目标的情况下，人仍然反复偏离？**

## 二、理论基础

Aligner 建立在三个稳定理论框架之上：

1. **Self-Monitoring**：有效改变依赖持续记录，尤其是时间、情境、情绪变量。
2. **EMA（生态瞬时评估）**：偏航往往源于当下情境与生理状态。
3. **BCT 行为改变组件**：目标设定、反馈、问题识别、复盘与纠偏构成有效闭环。

产品方法论整合为：**S-PDCA：State → Plan → Do → Check → Act**

## 三、目标人群与核心场景

Aligner 面向"明明知道该怎么做，却总是做不到"的人群——理性、上进、有目标的知识工作者或管理者。他们具备充足的信息储备（热量赤字、自律重要性、长期健康价值），但现实生活中的压力、疲劳、情绪波动和社交环境持续打断其节奏。

减脂作为入口场景，因饮食和身体状态每天都在发生、决策频繁、反馈明确，是训练"内在一致性"的理想领域。

Aligner 帮助用户在忙碌与压力之中，逐渐形成稳定的身份：**一个在复杂环境中仍能做出清晰选择的人**。服务对象为长期自我成长人群，而非短期减重需求。

## 四、系统逻辑

### 1. State（调频优先）

在高压力或疲劳状态下，系统优先触发微干预，使用户恢复可决策状态。

原则：**State Before Strategy**。

### 2. Plan（意图对齐）

将长期目标拆解为当日可执行边界。强调"身份一致性"，而非刚性数字。

### 3. Do（行为记录）

所有行为以"事件文件"形式记录，包含：时间戳、原始证据（图片/语音）、结构摘要、区间估算、情境标签与置信度。

### 4. Check（模式识别）

分析摄入区间趋势、情境变量、睡眠影响与连续性偏航。

关键问题：**偏航发生前的触发条件是什么？**

### 5. Act（结构纠偏）

系统调整策略、建议环境改变、更新观测维度。**阻抗被视为信息，而非错误。**

## 五、数据结构

Aligner 采用行为文件系统，而非传统数据库：

- 每日为目录
- 每次行为为带时间戳的事件文件
- 原始证据永久保存
- 结构层可演化

派生层生成趋势、风险预测与一致性指数。Coach Agent 拥有指标治理能力，可新增或退役观测维度。

## 六、Agent 架构与运作方式

用户只面对一个"教练"。教练负责倾听、观察上传的照片、理解当天发生的事情，并决定何时聊天、何时记录、何时提醒。

系统不对每句话过度反应。仅在与目标相关时整理信息，并结合近期数据做出判断。系统记住每天发生的事情，但不以复杂数字施压，而是关注趋势与模式。所有分析在后台完成，用户只面对一个冷静、稳定的教练。

核心：**让改变成为持续运行的过程，而非靠意志力硬撑的冲刺。**

## 七、差异化定位

Aligner 是长期运行的行为对齐系统，而非问答工具。核心特征：

- 状态优先的决策逻辑
- EMA 情境变量纳入
- 阻抗解码机制
- 可演化数据结构
- 连续性身份生成

**Aligner 生成一致性。**

## 八、愿景

减脂是起点。Aligner 本质上是一个**行为对齐操作系统**。

可扩展领域：

- 运动训练
- 睡眠优化
- 情绪管理
- 学习系统
- 创业决策

核心始终不变：**长期对齐，而非短期动力。**

当用户不再依赖意志力，而是通过结构化协议自然行动时，身份便开始演化。
