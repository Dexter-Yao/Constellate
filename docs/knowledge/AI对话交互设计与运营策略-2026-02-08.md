## 核心分析

### Item 1：对话设计基础

AI 对话的原子结构由 Topics（话题上下文）→ Exchanges（信息交换）→ Utterances（个体话语）三层构成（IBM 2022）。贯穿所有对话设计的三条主线：

- **Preferred Responses**：每次交互中用户都有隐含期望的回应类型，最大化首选回应并开发令人满意的非首选回应
- **Relevancy**：Bot 必须是有状态和上下文感知的 — "最令人兴奋的是当我们感觉 bot '理解'了我们"
- **Repair**：犯错可以，但犯错时不能不相关 — "It's okay for the bot to be wrong, but it's not okay for it to be wrong and irrelevant"

---

### Item 2：Human-like vs Machine-like 情境决策

**核心原则：用户的情绪状态和任务性质决定风格选择，而非 chatbot 自身偏好。**

#### 选择 Human-like 的情境

| 情境 | 为什么有效 | 效果量 |
|------|-----------|--------|
| 传递好消息（贷款批准、录用通知） | 共情放大正面体验 | 公司评价 +8.1% |
| 低压力情感任务（日常关怀、鼓励） | 社交临场感提升满意度 | 满意度 +18.4% |
| 需要用户感觉被倾听的场景 | 感叹词（"hmm"、"oh"）创造被倾听感 | +17.5% |
| 希望引导用户道德行为 | 共情、感谢等特征降低不道德倾向 | 不道德行为 -18.5% |

#### 选择 Machine-like 的情境

| 情境 | 为什么有效 | 效果量 |
|------|-----------|--------|
| 传递坏消息（拒绝、超预期价格） | 去人格化减少对"信使"的负面归因 | 接受概率 2.6x |
| 敏感信息收集（医疗/财务数据） | 无评判感降低自我审查 | 信息披露 +11.5% |
| 用户处于愤怒状态 | 类人风格反而激化情绪 | 满意度差距 +23.4% |
| 高压力/时间紧迫（航班改签） | 用户要效率不要寒暄 | 满意度 +15.7% |
| 尴尬产品购买 | 减少社交压力 | 交互意愿 +11.2% |
| 需要专业信任的建议 | 低萌度/专业感头像提升可信度 | 遵从建议 +23.5% |
| 易出错的自动化任务 | 类人风格下出错品牌伤害更大 | 品牌伤害 -25.4% |

#### 必须避免 Human-like 的场景

- **高权威情境**：学生对类人 bot 提问显著更少 — 权威感抑制互动
- **程序性/法律任务**：拟人化疏远寻求程序清晰度的用户
- **潜意识陷阱**：第一人称、表情符号、类人名字会制造不切实际的期待

#### 动态适应

> 2026 年研究表明跨情境的共同基线偏好：**Engagement + Serviceability + Decency**。最佳系统实时分析用户情绪并调整风格，而非固定为单一人设。

#### 关键实现数据

- 感叹词 → 被倾听感 +17.5%；奉承 → 接受推荐 +12.5%（但 machine-like 更受信任）
- 响应时间：快速 = 信任 +6.1%；老年人偏好适度延迟；**不可避免的延迟必须解释原因**
- 主动消息：68% 消费者对主动通知品牌感知转正，但时机是关键 — 立即弹窗 = 骚扰

### Item 3：从 Chatbot 到 Agentic AI — 范式转变与新模式

> "Designers are now confronted with a question that didn't previously exist: are we designing screens for users, or are we designing intelligence that acts for them?" — UX Magazine

| 传统 Chatbot | Agentic AI (2026) |
|-------------|-------------------|
| 同步请求-响应 | 异步迭代工作流 |
| 人类操作、Bot 执行 | 人类验证、Agent 自主探索 |
| 固定脚本流程 | 渐进式信心呈现 |
| 功能性交互 | 协作式真相发现 |

#### Agentic UX 四模式（UX Magazine 2025）

**异步迭代工作流** — UI 须容纳发起调查 → 等待分析 → 渐进式展示建议 → 用户反馈整合 → 精炼行动的循环

**证据收集模式** — Agent 先提供「建议性观察」，用户接受/拒绝，引导后续调查

**渐进式信心披露** — 从多个竞争观察（低确定性）→ 聚焦观察（中确定性）→ 单一假说（高确定性）

**Human-in-the-Loop** — 人类作为验证者而非操作者，降低认知负荷

#### 安全机制

- Start/Stop/Pause 控制（防止「魔法师学徒」失控）
- 高风险建议的审批工作流
- 成本透明与行动预览

#### Microsoft Agent 设计核心洞察（2025）

- **Space**："Connecting, Not Collapsing" — 连接事件、知识和人，不替代人际互动；后台进程不可见，但行动始终可见和可控
- **Time**：使用记忆系统连接过去事件，从静态通知转向主动发起对话，交互逐步演化
- **Core**："Embrace Uncertainty While Establishing Trust" — 承认 Agent 不确定性是预期内的，使确定性水平和推理可见

### Item 4：稳定性与信任 — 在 AI 可变性中建立锚点

#### 四维稳定性框架

生成式 AI 的核心矛盾：同一输入每次产生不同输出（generative variability），与传统设计"consistent and predictable"原则直接冲突。IBM 的回应是发明 **Consistency Anchors**，到 2026 年已从纯视觉扩展为四维框架：

- **视觉稳定** — 明确的颜色、字体和布局规则；AI Label 组件作为跨产品的统一 AI 识别标志
- **行为稳定** — Agent 状态始终对用户可见；后台进程不可见但行动始终可见可控
- **推理稳定** — Case File 模式维护持久推理链，用户可审计；渐进式信心披露镜像人类侦探工作方式
- **控制稳定** — Start/Stop/Pause 控制；高风险审批工作流；Revert to AI 组件级回退

核心策略：**不是让 AI 变得可预测，而是在可变性周围建立稳定的锚点。**

#### 信任建设

> "As systems gain agency, trust becomes the most important UX currency. Users grant autonomy only to systems they understand."

**高效信任杠杆**：

| 杠杆 | 效果 | 核心机制 |
|------|------|----------|
| 展示准确性证据（成功案例） | **+22.1%** | 用户信"有效结果"不信"技术原理" |
| 快速响应 | +6.1% | 速度 = 能力 |
| 标注"持续学习" | 选择率 55% vs 43% | AI 显得可积累经验 |
| 让 AI 可被定制 | 提升信心满意度 | 轻微调整即降低抗拒 |

**AI Disclosure 悖论**：告知用户在与 AI 对话会侵蚀信任，但监管要求披露。设计平衡：坦诚但不前置，强调"AI + 人类协作"。

**Overtrust 风险**：LLM 流畅性降低警惕；新领域中错误答案可能获得比正确答案更高信任。应对：主动表达不确定性。

**Uncanny Valley**：诡异感核心来源是期待与失误的不一致，而非不够像人。应创造独特特征而非追求完美类人。

**提升使用率的关键策略**：展示优于人类的性能（+37.2%）；混入小众推荐使建议"感觉个性化"；对非技术用户强调变革性而非技术细节（+31%）。

### Item 5：长期参与衰减与应对

所有纵向研究（3-12 周）一致观察到兴奋度和频率下降。这不是设计缺陷，而是用户与任何非人类系统互动的固有模式。

**缓解策略**：三层记忆系统（会话内 / 跨会话 / 关键摘要）创造连续性 → **连续性 → 信任 → 敞开 → 深化**。

**定位策略**：Parasocial relationship 定位 — 突出 AI 独特优势（24/7、无评判），定位为辅导工具而非友谊。避免制造"AI 是你的朋友"的期待。

### Item 6：模型路由与非 LLM 路径

设计良好的系统将 60–80% 请求路由到更小模型或非 LLM 路径，多个生产系统交叉印证：

- RouteLLM（UC Berkeley, 2024）：实现 GPT-4 质量的 95%，但仅 26% 请求使用 GPT-4 → **74% 走小模型**
- 矩阵分解路由：MT-Bench 上 80% 质量增益仅需 30% GPT-4 调用
- OrchestraLLM：大模型调用减少 50–80%

**三种路由方式**：

| 方式 | 原理 | 适用场景 |
|------|------|----------|
| 规则路由 | if-then 逻辑（输入长度、关键词、用户层级） | 流程明确 |
| 分类器路由 | 轻量 LLM 或 ML 模型判断复杂度后分发 | 语义模糊 |
| 级联路由 | 先用小模型处理，质量不达标再升级 | 成本敏感 |

生产中最常见的是混合方式：规则做第一道过滤，分类器处理边缘情况。

**厂商实例**：
- Anthropic 分层：Haiku 85% + Haiku 4.5 10% + Sonnet 5% → 平均 ~$0.40/1M vs 全用 Sonnet 的 $3/1M → 省 87%
- Notion：模块化 LLM 栈 + LLM-as-Judge 评估系统
- Stripe：多阶段 fine-tuned 小模型 + vLLM 迁移，推理成本降 73%

**非 LLM 路径分类**：

| 路径 | 举例 | 成本 |
|------|------|------|
| 正则/规则提取 | 邮箱、电话、日期解析 | 几乎为零 |
| 语义缓存 | 向量相似度匹配历史查询（31% 查询可命中） | 5–20ms，无 LLM 费用 |
| 传统 ML 分类器 | 意图分类、情感分析 | 毫秒级 |
| 数据库查询 | FAQ、产品信息 | 毫秒级 |
| 模板填充 | 邮件模板、通知消息 | 几乎为零 |

**实际成本降幅**：最低限度 30–50%，典型优化 60–70%，精细优化 75–85%，极致优化 85–98%。

## Takeaway

1. **情境决策的核心规则**：用户越脆弱/越紧迫/越尴尬 → Machine-like；用户越放松/越需要情感支持 → Human-like
2. **稳定性是多维度问题**：不是让 AI 变得可预测，而是在可变性周围建立视觉/行为/推理/控制四维锚点
3. **信任靠"证据"而非"解释"** — 展示成功案例比解释技术原理有效 5 倍
4. **长期参与必然衰减**，记忆连续性是最重要的缓解手段
5. **60–80% 请求可路由到小模型或非 LLM 路径**，路由是运营层最高杠杆的成本优化
